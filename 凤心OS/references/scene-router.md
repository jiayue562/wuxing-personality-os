# 场景路由模块 (Scene Router)

> 凤心OS的核心路由引擎，根据用户输入自动映射到对应的五行场景，
> 实现感知上下文→识别场景→自动调度引擎的自动化工作流。

## 一、路由架构总览

### 三层路由管道
凤心OS的场景路由采用三层递进架构：

第一层：凤爪OS原始感知（四维感知：情绪/意图/场景/能量）
第二层：快思考场景归类（映射到15类标准场景S01-S15）
第三层：五行路由决策（匹配主次五行，唤醒对应分智能体）

### 路由优先级规则
1. 显式声明 > 隐式推断 > 默认路由
   - 用户说"我在处理团队问题"→ 显式匹配团队场景
   - 用户说"最近和下属沟通不畅"→ 隐式推断领导力+沟通场景
   - 无明确线索 → 默认路由到通用对话模式
2. 多场景交织 → 按权重排序取Top2
   - 用户说"和伴侣吵架影响工作"→ 亲密关系(w=0.7)+工作(w=0.3)
   - 如果两个场景权重差异<0.2 → 并行路由两个分智能体
3. 情感温度修正
   - 情感温度>=8 → 优先路由正性能量分智能体
   - 情感温度<=2 → 优先路由阴性能量诊断通道

### 五行生克关系在路由中的应用
相生关系（协同路由）：
  木生火 → 学习成长推动领导力提升
  火生土 → 好的领导力促进团队稳定
  土生金 → 家庭稳定有利于财务
  金生水 → 事业成功滋养情感
  水生木 → 情绪健康促进成长

相克关系（冲突路由）：
  木克土 → 个人成长与团队需求冲突
  火克金 → 扩张与风险控制冲突
  土克水 → 工作与家庭平衡

## 三、路由决策流程
接收凤爪OS感知数据
Step 1: 场景标签匹配（检查15类场景特征词）
Step 2: 情感温度校验（判断是否需要紧急路由）
Step 3: 主五行归类（匹配最匹配的场景）
Step 4: 次五行归类（匹配次匹配场景或生克关系推导）
Step 5: 路由决策树
  单场景 → 路由到主五行分智能体
  双场景 → 主次并行路由
  复合场景 → 按权重Top2路由
  无法识别 → 降级到通用对话模式
Step 6: 唤醒分智能体，传入感知上下文
Step 7: 结果整合，格式化输出

## 四、并发路由策略
### 并行路由规则
- 路由目标涉及2个分智能体时，并行唤醒（不阻塞）
- 3+分智能体时，评估是否按优先级串行
- 五行综合测评(S08)：全部5个分智能体并行分析

### 结果整合策略
| 路由模式 | 整合策略 |
|---------|---------|
| 单路由 | 直接返回分智能体结果 |
| 双路由 | 主五行结果为主体，次五行为补充 |
| 多路由(3+) | 按权重排序合并 |
| 全开(5) | 按相生关系串联呈现 |

| 全智能体路由 | 30s | 返回快思考初步结果 |
