# 快思考-慢思考引擎 (Fast-Slow Thinking)

> 凤心OS的双系统思维架构，实现1+5模式的智能调度

## 快思考系统（System 1）
- 响应时间：<500ms
- 功能：场景理解、五行归类、引擎路由
- 特征：直觉性、自动化、低能耗
- 触发条件：所有输入默认先过快思考

## 慢思考系统（System 2）
- 响应时间：2-30s
- 功能：深度分析、生克推演、方案生成
- 特征：分析性、可控性、高能耗
- 触发条件：快思考判断复杂度 > 阈值，或用户主动请求深度分析

## 调度逻辑
1. 输入 → 快思考（场景识别+五行归类）
2. → 路由决策（是否需要慢思考？）
3. → 慢思考（分智能体深度处理）
4. → 结果整合与输出

## 能量分配
- System 1 处理80%日常场景
- System 2 处理20%复杂场景
- 复杂场景自动唤醒关联分智能体并行处理
