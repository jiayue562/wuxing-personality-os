# 快思考-慢思考引擎 (Fast-Slow Thinking)

> 凤心OS的双系统思维架构，实现1+5模式的智能调度。
> 基于卡尼曼《思考，快与慢》理论，适配五行人格心理学OS的个性化改造。

## 一、双系统理论概述

### 快思考系统（System 1）——直觉引擎
凤心OS的快思考系统是默认入口，所有用户输入首先经过快思考处理。

| 属性 | 指标 |
|------|------|
| 响应时间 | <500ms |
| 能耗 | 低（约占CPU时间20%） |
| 处理量 | 日常场景的80% |
| 处理方式 | 直觉并行处理 |
| 意识层级 | 自动化、无意识 |

**快思考的三层感知雷达**：
1. 语境雷达：识别用户输入的场景语境
2. 情绪雷达：感知用户情感温度和能量状态
3. 意图雷达：推断用户底层需求与动机

**快思考五行动作序列**：输入→语境识别→情感温度标记→意图推断→五行归类→路由决策

### 慢思考系统（System 2）——深度分析引擎
当快思考判断场景复杂度超过阈值时，唤醒慢思考系统。

| 属性 | 指标 |
|------|------|
| 响应时间 | 2-30s |
| 能耗 | 高（约占CPU时间80%） |
| 处理量 | 复杂场景的20% |
| 处理方式 | 分析串行处理 |
| 意识层级 | 受控、有意识 |

**慢思考五层深度处理管道**：
1. 精确诊断：调用L7理论基石进行五行人格精确诊断
2. 生克推演：基于五行生克关系进行动态推演
3. 跨域联系：调用凤脑OS知识图谱中的130+条跨域联系
4. 方案生成：生成个性化的五行平衡方案
5. 结果整合：将分析结果格式化为用户可读的输出

## 二、快慢切换机制

### 自动触发条件
| 条件 | 触发值 | 动作 |
|------|--------|------|
| 情感温度过高 | >=8 或 <=2 | 唤醒对应分智能体 |
| 场景复杂度高 | 3+场景交织 | 唤醒慢思考 |
| 五行冲突明显 | 2+五行要素冲突 | 并行唤醒相关智能体 |
| 用户深度提问 | 含为什么/怎么办/分析等词 | 唤醒慢思考 |
| 历史模式匹配 | 该用户有深度请求记录 | 预唤醒慢思考 |

### 手动触发
用户要求深度分析、诊断五行、帮我推演时，直接进入慢思考。

### 能量分配
| 级别 | 占比 | 处理方式 |
|------|:----:|---------|
| 日常对话 | 80% | 仅快思考，返回即时反馈 |
| 轻度分析 | 15% | 快思考+指定分智能体 |
| 深度推演 | 5% | 快思考+慢思考（1+N全开） |

## 三、1+N 并行调度模式
凤心OS采用 1个快思考总调度 + N个慢思考分智能体的并行模式。

### 智能体唤醒矩阵
| 场景 | 唤醒分智能体 | 协作模式 |
|------|-------------|---------|
| 亲密关系冲突 | 水+火 | 水火既济 |
| 事业决策困难 | 金+水 | 金生水 |
| 团队管理难题 | 金+土 | 土生金 |
| 个人成长迷茫 | 木+火 | 木生火 |
| 健康调理 | 土+水 | 土克水平衡 |
| 五行综合测评 | 全部5个 | 全开并行 |

## 四、错误处理与降级机制
| 层级 | 故障场景 | 降级策略 |
|------|---------|---------|
| 快思考 | 超时(>1s) | 回复请稍等 |
| 快思考 | 识别失败 | 切换通用关键词匹配 |
| 慢思考 | 单智能体超时(>30s) | 跳过该智能体 |
| 慢思考 | 全部超时 | 降级为快思考结果 |
| 知识库 | 凤脑OS离线 | 降级模式（仅内置知识） |
